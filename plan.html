Who owns what

Roles & deliverables

Person 1

🏗️ Harness Architecture & OpenCode Integration

Owns the critical path: the module contracts everyone else builds against.

Week 1
  • Design the three-module abstraction & storage interface (swappable backends).
  • Build the persistence layer: write logic, versioning, metadata tagging.
  • Instrument OpenCode to write memory on each agent step.
Week 2
  • Build the retrieval orchestrator (rank → dedup → return).
  • Wire in Person 3's router; run an end-to-end task.
Person 2

📊 Benchmarking & Evaluation

Owns experimental design, metrics, and reproducibility.

Week 1
  • Download & parse MemoryAgentBench, LongMemEval, SWE-ContextBench, SWE-Bench-CL.
  • Build data loaders + trajectory logging (what was written/retrieved/used).
  • Define the four metrics mathematically.
Week 2
  • Run baselines: Haiku no-memory, Opus 4.8 no-memory.
  • Statistical testing framework + results dashboard; document the protocol.
Person 3

🗄️ Storage Implementation & Router

Turns the abstraction into three real, fast backends + the dispatch layer.

Week 1
  • SQLite + vector pipeline (embedding model, HNSW/FAISS index).
  • Markdown store with YAML frontmatter + inverted index.
  • Graph store schema (Neo4j) with typed traversal index.
Week 2
  • Adapters for all three backends behind Person 1's interface.
  • Implement the router; performance-test each backend.
Person 4

🌙 Zooming Component & Memory Governance

Owns the offline engine that keeps memory clean and trustworthy.

Week 1
  • Deduplication logic (exact / semantic / near-duplicate).
  • Conflict detection + reconciliation rules (recency, confidence, source).
  • Session-filter & blacklist semantics.
Week 2
  • Async offline scheduler; must-know / must-do extraction.
  • Retention & pruning; observability logs; integration with P1 & P3.

Dependencies

⚠️

The hardest risk is semantic, not structural: what counts as "good memory." Person 2 and Person 4 must align on memory semantics in week 1, or the harness will store everything and retrieve nothing useful.

  • P1 → P3: the storage interface must be frozen by end of D3 so adapters can be built against it.
  • P3 → P1: the router lands before the orchestrator can route in week 2.
  • P2 → all: datasets + logging ready by D5 so baselines can start D6.
  • P1 + P3 → P4: the zooming worker integrates once real storage exists (D8+).
