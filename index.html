Persistent memory that lets a small model think like a big one.

A pluggable memory harness for long-running coding agents. It decides what to remember, where to store it, and how to retrieve it fast — then quietly cleans itself up offline. The thesis: with the right memory, Haiku can close the gap to Opus 4.8 on memory benchmarks.

The hypothesis

Memory is leverage. We want to measure exactly how much.

Frontier models win partly because they hold more in their head at once. If an agent can reliably save, retrieve, and reconcile what it learns across sessions, a cheaper model should be able to borrow that advantage.

🎯

Success criterion

On at least two memory benchmarks, Haiku + the memory harness beats Opus 4.8 with no memory — measured across recency, efficiency, relevancy, and accuracy. The harness is model-agnostic and is validated against existing public benchmarks, not ones we invented.

How it works

Four modules, one loop

A coding agent writes memories as it works. A router sends each query to the one backend that answers it best. An offline worker keeps the whole store clean and trustworthy.

Module 1 · Write

💾 Persistence Layer

Decides what, when and where to save. Each memory item is tagged with a timestamp, a relevancy score and a session tag, then written to one or more of the three backends.

Module 2 · Dispatch

🧭 Intelligent Router

Instead of fanning out to all three stores, the router classifies the query and picks the single best backend — graph for relationships, vectors for semantics, keywords for Markdown. Rule-based to start, learnable over time.

Module 3 · Read

🔎 Retrieval Orchestrator

A unified query API. It calls the router, ranks results by recency × relevancy, de-duplicates, and returns a tight context the agent can actually use.

Module 4 · Curate (async)

🌙 Zooming Component

Runs offline when the agent is idle. It de-duplicates, resolves contradictions, and maintains session governance — what must be known, what must be done, and what is blacklisted forever.

What we measure

The four memory metrics

⏱️

Recency

Temporal distance of retrieved memory to the current task. Is the freshest relevant item ranked first?

⚡

Efficiency

Tokens spent per retrieval. Target: memory adds less than ~10% context overhead.

🎯

Relevancy

Semantic similarity of retrieved items to the query. Only pull memories that truly relate.

✅

Accuracy

Correctness of agent output with memory vs. without. The bottom-line measure of value.

Validated against

Four public benchmarks — no home-grown evals

MemoryAgentBench

Accurate retrieval · test-time learning · long-range understanding · conflict resolution.

LongMemEval

Temporal reasoning, knowledge updates and abstention across long, timestamped histories.

SWE-ContextBench

Context reuse across related coding tasks — 51 repos, 9 languages, accuracy & cost.

SWE-Bench-CL

Continual learning: knowledge transfer and resistance to catastrophic forgetting.
