The hypothesis

Memory is leverage. We measure how much.

Frontier models win by holding more in their head. Give a smaller model a memory it can save, retrieve, and reconcile — and it borrows that edge.

🎯

Success criterion

On ≥2 benchmarks, Haiku + harness beats Opus 4.8 with no memory across recency, efficiency, relevancy, accuracy — on public benchmarks, not ones we invented.

How it works

Four modules, one loop

The agent writes memories as it works. The router sends each query to the one backend that answers best. An offline worker keeps the store clean.

Module 1 · Write

💾 Persistence Layer

Decides what, when, where to save. Tags each item (timestamp, relevancy, session) and writes it to the right backend(s).

Module 2 · Dispatch

🧭 Intelligent Router

Classifies the query and hits the one best backend — graph for relationships, vectors for semantics, keywords for Markdown. Rule-based, then learned.

Module 3 · Read

🔎 Retrieval Orchestrator

Unified query API: route, rank by recency × relevancy, de-dup, return a tight context.

Module 4 · Curate (async)

🌙 Dreaming Component

Runs while other agents sleep: de-dup, resolve contradictions, set session governance (must-know / must-do / blacklist).

What we measure

The four memory metrics

⏱️

Recency

Is the freshest relevant memory ranked first?

⚡

Efficiency

Tokens per retrieval — target < ~10% overhead.

🎯

Relevancy

Retrieved items actually relate to the query.

✅

Accuracy

Task correctness, memory on vs. off.

Validated against

Five public benchmarks — no home-grown evals

MemoryAgentBench

Accurate retrieval · test-time learning · long-range understanding · conflict resolution.

LongMemEval

Temporal reasoning, knowledge updates and abstention across long, timestamped histories.

SWE-ContextBench

Context reuse across related coding tasks — 51 repos, 9 languages, accuracy & cost.

SWE-Bench-CL

Continual learning: knowledge transfer and resistance to catastrophic forgetting.

ContextBench

In-task retrieval quality: gold contexts (file/block/line), recall · precision · efficiency.
