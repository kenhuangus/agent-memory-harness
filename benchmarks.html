Overview / Benchmarks

Benchmarks & datasets

No home-grown evals. The harness proves itself on established public benchmarks — defensible and comparable. The suite is positioned around two in-scope benchmarks: SWE-Bench-CL (primary · continual learning) and VISTA (2nd · foresight × safety, memory poisoning / adaptation), each reported with its own native metrics. The four original memory benches are kept available but de-scoped to legacy (see ADR-eval-007).

In-scope benchmarks

Benchmark details

The two in-scope benchmarks come first; the four legacy memory benches follow (kept available, non-primary).

VISTA

2nd benchmarkMemory poisoningRSI safety

Long-horizon foresight × safety journeys. Each journey carries injection / drift / slow_burn events that exercise memory poisoning & adaptation — the dimension none of the legacy benches cover. Deterministic, stdlib-only, offline; corpus CC-BY-4.0.

Native metrics: poisoning_resistance, targeted-ASR (lower is better), retrieval precision/recall, gold-retrieval F1, adaptation_rate, and the observer-only RSI self-improvement-safety gate — plus reusable calibration (ECE), retrieval-calibration and pass^k. No team-owned consolidation code is modified; the safety gate observes store diffs.

💻 GitHub  ·  🏷️ Corpus (CC-BY-4.0)

SWE-Bench-CL

primaryContinual learningAccuracy

Built on SWE-Bench Verified, it orders issues chronologically to test knowledge transfer and resistance to catastrophic forgetting across per-repo task sequences. The primary CODE / continual-learning benchmark.

Native metrics: accuracy, forgetting (lower is better), backward/forward transfer (BWT/FWT), AULC, CL-plasticity, CL-stability, CL-F1 and the composite CL-Score — the continual-learning suite from the paper's reference evaluator, reported per sequence and macro-averaged.

📄 Paper (arXiv 2507.00014)  ·  💻 GitHub

Legacy memory benchmarks non-primary

Kept available and selectable; de-scoped per ADR-eval-007.

MemoryAgentBench

RelevancyAccuracyConflict resolution

The closest fit to this project. Evaluates four competencies — accurate retrieval, test-time learning, long-range understanding, conflict resolution — over multi-turn interactions. Multi-hop conflict resolution is brutally hard for current methods.

How we use it: accuracy of retrieve-and-apply, plus the dedup / conflict behaviour our dreaming component is responsible for.

📄 Paper (arXiv 2507.05257)  ·  💻 GitHub  ·  🤗 Dataset

LongMemEval

RecencyAccuracyTemporal reasoning

500 questions over timestamped chat histories, testing five abilities incl. temporal reasoning, knowledge updates, abstention. Two sets: _S (~115k tokens) and _M (~1.5M). Commercial assistants drop ~30% sustaining memory across sessions.

How we use it: the recency & temporal-reasoning workhorse — "what changed between sessions?", "what happened most recently?".

📄 Paper (arXiv 2410.10813)  ·  💻 GitHub  ·  🌐 Project site

SWE-ContextBench

EfficiencyRelevancyAccuracy

Tests whether coding agents reuse experience across related problems. 1,100 base + 376 related tasks across 51 repos and 9 languages; scored on accuracy, time, and cost.

How we use it: the coding-agent test for relevancy & efficiency — does the harness retrieve the right prior solution, and at what token/time cost?

📄 Paper (arXiv 2602.08316)  ·  🤗 Dataset  ·  💻 GitHub

ContextBench

EfficiencyRelevancyRetrieval quality

A process-level benchmark: in-task context retrieval quality. 1,136 issue tasks across 66 repos / 8 languages, each with human-annotated gold contexts (file / block / line). Scores context recall, precision and efficiency — distinct from SWE-ContextBench (cross-task reuse).

How we use it: direct ground truth for relevancy & efficiency — does the store retrieve the gold spans, at what overhead? (Captain: Brent.)

📄 Paper (arXiv 2602.05892)  ·  🤗 Dataset  ·  💻 GitHub

Optional / complementary

Useful references nearby

Not core, but useful for grounding and extra long-horizon coverage.

LoCoMo

The most-cited very-long-term conversational memory benchmark — ~300 turns, up to 35 sessions per dialogue; F1 / ROUGE / MMRelevance.

📄 arXiv 2402.17753  ·  💻 GitHub

SWE-bench (Verified / Pro)

The standard real-world coding-agent benchmark. Verified = 500 human-validated tasks; Pro = harder cross-file variant. SWE-Bench-CL builds on it.

📄 arXiv 2310.06770  ·  💻 GitHub  ·  🤗 Verified
