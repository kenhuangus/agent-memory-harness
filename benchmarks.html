Benchmarks & datasets

We do not build our own evals. The harness proves itself on established, public benchmarks that explicitly measure memory — so the results are defensible and comparable.

The four anchors

Benchmark details

MemoryAgentBench

RelevancyAccuracyConflict resolution

The benchmark most aligned with this project. It evaluates agent memory across four core competencies — accurate retrieval, test-time learning, long-range understanding, and conflict resolution — by restructuring datasets into multi-turn incremental interactions. Includes two purpose-built datasets, EventQA and FactConsolidation, that expose where memory systems break (multi-hop conflict resolution is brutally hard for current methods).

How we use it: accuracy of retrieve-and-apply, plus the dedup / conflict behaviour our zooming component is responsible for.

📄 Paper (arXiv 2507.05257)  ·  💻 GitHub  ·  🤗 Dataset

LongMemEval

RecencyAccuracyTemporal reasoning

500 curated questions over freely scalable, timestamped chat histories, testing five long-term memory abilities: information extraction, multi-session reasoning, temporal reasoning, knowledge updates, and abstention. Commercial assistants show a ~30% accuracy drop sustaining memory across sessions. Two test sets: LongMemEval_S (~115k tokens / question) and LongMemEval_M (~1.5M tokens).

How we use it: the recency & temporal-reasoning workhorse — "what changed between sessions?", "what happened most recently?".

📄 Paper (arXiv 2410.10813)  ·  💻 GitHub  ·  🌐 Project site

SWE-ContextBench

EfficiencyRelevancyAccuracy

Tests whether coding agents reuse experience across related problems. 1,100 base tasks plus 376 related tasks derived from real dependency/reference relationships among GitHub issues and PRs, grouped with shared context across 51 repositories and 9 programming languages. Scored on three dimensions: prediction accuracy, time efficiency, and cost efficiency.

How we use it: the coding-agent test for relevancy & efficiency — does the harness retrieve the right prior solution, and at what token/time cost?

📄 Paper (arXiv 2602.08316)  ·  🤗 Paper page

SWE-Bench-CL

RelevancyAccuracyContinual learning

A continual-learning benchmark built on SWE-Bench Verified. It organizes GitHub issues into chronologically ordered sequences to test whether an agent can accumulate experience, transfer knowledge, and resist catastrophic forgetting. Ships a LangGraph evaluation framework with a FAISS-backed semantic memory module and CL metrics: average accuracy, forgetting, forward/backward transfer, tool-use efficiency, and a composite CL score.

How we use it: relevancy precision & accuracy over time — avoid negative transfer (irrelevant memories that hurt) across task sequences.

📄 Paper (arXiv 2507.00014)  ·  💻 GitHub

Optional / complementary

Useful references nearby

Not in the core four, but valuable for grounding the design and for extra long-horizon coverage.

LoCoMo

The most-cited very-long-term conversational memory benchmark — ~300 turns and up to 35 sessions per dialogue, scored with F1, ROUGE and MMRelevance over QA, summarization and multimodal generation.

📄 arXiv 2402.17753  ·  💻 GitHub

SWE-bench (Verified / Pro)

The standard real-world coding-agent benchmark. Verified is a 500-task human-validated subset; Pro is the harder, long-context, cross-file variant. The accuracy anchor SWE-Bench-CL is built upon.

🌐 swebench.com  ·  💻 GitHub  ·  🤗 Verified
