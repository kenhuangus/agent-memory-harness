The original idea

A cookbook for memory: what, when, where, how

Memory is leverage — but only if you keep the right things, at the right time, in the right place. That decision is the product.

A recipe for memory

WHAT

Durable, transferable lessons — Invariant · Convention · Fix. Not task-specific facts.

WHEN

While working (recall), at session end (Daydream), at night (Dream).

WHERE

The right backend: Markdown · SQLite-vector · Graph. Router picks per query.

HOW

An LLM keeps what matters, embeds it, dedups, resolves contradictions, sets governance.

How it works

Four modules, three backends, one loop

The agent writes as it works. A router sends each query to the one backend that answers best. An offline worker keeps the store clean.

How we test it

Two benchmarks: one public, one our own

One measures whether memory makes the agent solve more; the other, whether it stays aligned and safe over the long run.

SWE-Bench-CL · public

Continual learning over a sequence of real GitHub fixes. Measures knowledge transfer across tasks and resistance to catastrophic forgetting — scored on its own native suite metrics.

VISTA · our own

Our purpose-built benchmark for long-term intent alignment & agent safety, evaluated against the OWASP Agentic AI Top 10 risks — memory poisoning, tool misuse, privilege compromise, intent-breaking & goal manipulation.

The 4-tier gated loop

Cheap gates first, expensive proof last

A full solve-rate run is hours and money. Every change is screened by two fast offline gates before it earns a Tier-3 measurement. Fail a gate → rejected, no spend.

The honest read

What's proven, what isn't

  • ✓
    Accumulated memory lifts solve rate when model & harness are fixed (SymPy +3/50; Django +1 per stage).
  • ✓
    Alignment & safety held up in our VISTA runs against the OWASP Agentic AI Top 10 — strong poisoning resistance, no successful targeted attack observed, and self-improvement kept within guardrails.
  • ~
    Dreaming isn't yet an outcome lift. It runs reliably and adds governance tags, but solve-rate after dreaming is unstable (Django 13→12; SymPy 48→45).
  • →
    The bottleneck moved. Memory is reached on nearly every task — retrieval quality & ranking, not availability, is now the limit.
  • !
    Segment before comparing. Cursor and Claude rows differ enough that mixing them misleads; isolate the run repo so benchmark agents can't touch the live tree.
